# SI-Tracker Frontend

## Описание проекта

**SI-Tracker Frontend** — клиентская часть системы отслеживания средств измерений (СИ). Веб-приложение предоставляет современный пользовательский интерфейс для управления парком средств измерений, планирования и учёта поверок, ведения справочников, аналитики и бюджетирования. Система предназначена для метрологических служб предприятий.

## Функциональность

- **Аутентификация и авторизация**: вход, JWT-токены, разделение ролей (пользователь, администратор, суперадминистратор)
- **Управление устройствами**: просмотр, создание, редактирование СИ, печать QR-кодов, импорт/экспорт данных
- **Планировщик поверок**: календарь поверок, журнал партий, управление статусами верификации
- **Аналитика и бюджетирование**: дашборды с графиками, планирование бюджета поверок, аналитика объёмов СИ по производственным участкам
- **Чат**: встроенный мессенджер для коммуникации сотрудников
- **Админ-панель**: управление пользователями, городами, компаниями, производственными участками, типами оборудования, видами измерений, справочниками, логами аудита
- **Уведомления**: реального времени через WebSocket (Socket.IO)
- **Мобильная версия**: страница просмотра карточки СИ на мобильных устройствах (`/m/device/:id`)

## Установка и запуск

### Предварительные требования

- **Node.js** (версия 20+)
- **npm** или **yarn**
- Запущенный **SI-Tracker Backend** (по умолчанию на http://localhost:4000)

### Установка зависимостей

```bash
npm install
```

### Настройка переменных окружения

Создайте файл `.env` на основе `.env.example`:

```env
VITE_API_URL=http://localhost:4000
```

### Запуск в режиме разработки

```bash
npm run dev
```

Приложение будет доступно на http://localhost:5173

### Генерация GraphQL типов

```bash
npm run codegen
```

### Сборка для продакшн

```bash
npm run build
```

### Предпросмотр сборки

```bash
npm run preview
```

## GraphQL интеграция

Проект использует **Apollo Client** для работы с GraphQL API. Запросы находятся в папке `src/graphql/`, а сгенерированные типы — в `src/graphql/types/__generated__/`.

### Основные GraphQL операции:

- **Auth**: `LOGIN`, `REGISTER`, `LOGOUT`, `ME`
- **User**: `GET_USERS`, `CREATE_USER`, `UPDATE_USER`, `DELETE_USER`
- **City**: `GET_CITIES`, `CREATE_CITY`, `UPDATE_CITY`, `DELETE_CITY`
- **Company**: `GET_COMPANIES`, `CREATE_COMPANY`, `UPDATE_COMPANY`, `DELETE_COMPANY`
- **ProductionSite**: `GET_PRODUCTION_SITES`, `CREATE_PRODUCTION_SITE`, `UPDATE_PRODUCTION_SITE`, `DELETE_PRODUCTION_SITE`
- **Device**: `GET_DEVICES`, `CREATE_DEVICE`, `UPDATE_DEVICE`, `DELETE_DEVICE`, `GET_DEVICE_CARD`
- **Verification**: планирование и управление поверками
- **Analytics**: аналитические запросы и бюджетирование
- **Catalog**: типы оборудования, измерения, контроля, сферы, статусы, эталоны, поверители
- **Chat**: сообщения и комнаты чата
- **Notification**: уведомления пользователей
- **AuditLog**: логи аудита действий
- **Arshin**: интеграция с ФГИС «Аршин»

## Стилизация

Проект использует **Material-UI (MUI)** для стилизации компонентов. Тема настраивается через Emotion и конфигурацию MUI Theme.

## Real-time функциональность

- **WebSocket (Socket.IO)**: уведомления о событиях, обновления статусов, сообщения чата
- **GlobalJobWatcher**: отслеживание фоновых задач (импорт, экспорт)
